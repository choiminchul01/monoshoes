대시보드 구성

  • 오늘의 매출: 당일 총 결제 금액

  • 최근 주문 목록: 최신 주문 현황 미리보기

주문 관리 핵심 기능

주문 상태 흐름

결제완료(Paid) → 배송준비(Preparing) → 배송중(Shipped) → 배송완료(Delivered) ↓ 취소(Cancelled) / 환불(Refunded)

주문 처리 절차

  1. 결제완료 확인: 새 주문이 "Paid" 상태로 표시됨
  2. 상품 준비: 재고 확인 후 포장 진행
  3. 배송 처리: "Shipped" 상태로 변경 + 송장번호 입력
  4. 배송완료: 배송 완료 확인 후 "Delivered"로 변경

※ 결제완료 시 주문 금액의 1% 포인트가 자동 적립됩니다.

주문 상세 정보

  • 주문자 정보 (이름, 연락처, 이메일)
  • 배송지 정보 (우편번호, 주소, 상세주소)
  • 주문 상품 목록 (옵션, 수량, 금액)
  • 결제 정보 (포인트 사용, 쿠폰 적용, 최종 금액)

쿠폰 관리

쿠폰 유형

유형 설명 예시
PERCENT 퍼센트 할인 10% 할인
FIXED 고정 금액 할인 5,000원 할인

쿠폰 설정 옵션

  • 쿠폰 코드: 고객이 입력할 코드 (예: WELCOME10)
  • 할인 값: 퍼센트 또는 금액
  • 최소 주문 금액: 쿠폰 사용 가능 최소 금액
  • 유효 기간: 시작일 ~ 종료일
  • 사용 횟수 제한: 전체/1인당 사용 제한

자동 발급 쿠폰

신규 가입 쿠폰 회원가입 시 자동으로 환영 쿠폰이 발급됩니다.

포인트 시스템

포인트 정책

항목 내용
적립률 결제 완료 시 결제 금액의 1% 자동 적립
사용 단위 1포인트 = 1원
사용 조건 결제 시 보유 포인트 한도 내에서 사용
유효 기간 적립일로부터 1년

배송 정책

배송비 안내

  • 기본 배송비: 관리자 설정에서 지정한 금액
  • 도서산간 추가: 도서산간 지역 추가 배송비
  • 무료배송: 기본 배송비 0원 설정 시

파트너 제휴

제휴 문의 프로세스

  1. 고객이 /partner 페이지 접속
  2. 제휴 제안서 이미지 확인
  3. /partner/inquiry에서 제휴 문의 양식 작성
  4. 관리자 문의 관리 페이지에서 확인 및 답변

기술 스택

Frontend

  • Framework: Next.js 15.5.7 (App Router)
  • Language: TypeScript
  • Styling: Tailwind CSS
  • Animation: Framer Motion

Backend

  • Database: Supabase (PostgreSQL)
  • Authentication: Supabase Auth
  • Storage: Supabase Storage
  • Hosting: Vercel

데이터베이스 구조

주요 테이블

users : 회원 정보 (Supabase Auth와 연동) products : 상품 정보 orders : 주문 정보 order_items : 주문 상품 상세 coupons : 쿠폰 정보 user_coupons : 사용자별 발급된 쿠폰 points : 포인트 잔액 point_transactions : 포인트 적립/사용 내역 wishlist : 위시리스트 reviews : 상품 리뷰 inquiries : 1:1 문의 product_qna : 상품 Q&A events : 이벤트/팝업 inspections : 출고 검수 admin_roles : 관리자 역할 및 권한 site_settings : 사이트 설정 banners : 메인 배너
